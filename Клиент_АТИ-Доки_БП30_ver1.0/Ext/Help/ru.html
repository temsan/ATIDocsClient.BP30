1. Общие сведения о Системе


Назначением 1С Обработки является создание, просмотр, утверждение/отклонение и подписание электронных документов в системе электронного документооборота АТИ-Доки.
1С Обработка может использоваться как замена веб-интерфейса системы электронного документооборота.
Кроме того, 1С Обработка позволяет формировать электронные документы из документов 1С и наоборот - формировать документы 1С из полученных электронных документов.

1.1. Функции системы

Система позволяет совершать следующие действия над документами:

  • создание;
  • поиск;
  • просмотр;
  • подписание;
  • отправка;
  • печать;
  • формирование в базе 1С;
  • утверждение;
  • отклонение;
  • удаление.

 

2. Порядок работы

2.1. Начало работы


1С Обработка адаптирована для конфигурации «1С: Бухгалтерия предприятия» редакция 3.0 и отраслевых решений на ее основе. Для корректной работы с электронными подписями (далее по тексту - ЭП) требуется версия платформы «1С: Предприятие» не ниже 8.3.4.

Открыть обработку в режиме «1С: Предприятие» можно двумя способами:

  • через меню «Файл».
  • через меню «Сервис».


2.1.1. Открытие 1С Обработки через меню «Файл»


Для открытия 1С Обработки в меню «Файл» выберите пункт «Открыть» (для этого у пользователя должны быть права на открытие внешних обработок).


2.1.2. Открытие 1С Обработки через меню «Сервис»


Для открытия 1С Обработки в меню «Сервис» выберите пункт меню «Дополнительные отчеты и обработки», зарегистрируйте обработку в служебном справочнике, затем выберите пункт «Дополнительные внешние обработки». В этом случае дополнительных прав на открытие внешних обработок не требуется.

 

2.2. Первичная настройка обработки


В форме 1С Обработки предварительно заполните следующие поля:

  • «Адрес сервера» - аналогично адресу при подключении через веб-интерфейс (например: «d.ati.su»);
  • «Логин», «Пароль»- учетные данные в системе АвтоТрансИнфо.
  • «Документов на странице» - количество одновременно отображаемых документов в выбранной папке. Не рекомендуется выбирать значение более 50, поскольку это может замедлить работу системы обновления данного списка.
  • «Каталог Агента» - каталог, выбранный в настройках служебной утилиты «Агент АТИ-Доки» (при ее использовании).


Для сохранения выбранных параметров, выберите пункт «Сохранить настройки».
Заполнив параметры, можно перейти на основную страницу. Для перехода на основную страницу нажмите кнопку «Применить».

В левой части панели отображается дерево папок с общим количеством документов. Рядом с наименованием папки в скобках отображается количество документов, требующих рассмотрения (принятия решения). Обратите внимание, что папка «Все документы» включает только входящие и исходящие документы и не включает документы в папках «Черновики», «Корзина» и «Архив».

Правая часть панели содержит таблицу со списком документов с основными атрибутами и доступными действиями.
При выборе строки таблицы доступность кнопок меняется в зависимости от свойств выбранного документа.
Также всегда доступно создание нового документа из меню «Действия» - «Отправить новый документ».

Поддерживается создание следующих типов документов:

  • Акт;
  • Доверенность;
  • Заказ (Договор-заявка);
  • Документ;
  • Другой;
  • Заказ услуги по оформлению пропусков в г. Москва;
  • Информационное письмо;
  • Накладная;
  • Счет;
  • Счет-фактура.

В зависимости от типа документа можно выбрать либо файл на диске, либо документ в базе 1С, на основании которого будет сформирован файл для электронного документа в соответствии с таблицей (см. Таблица 1. «Типы документов из базы 1С»).

При создании документа выберите адресата одним из двух способов:

  • При наличии связи (контрагент 1С добавляется автоматически) - укажите точное значение кода АТИ в поле «Получатель (код)».
  • При отсутствии привязки к коду АТИ - укажите наименование контрагента в поле «Контрагент (1С)». Данное поле поддерживает «нестрогий» поиск. Если по введенной строке найдено несколько аккаунтов, то отображается выпадающий список с вариантами выбора контрагента.

Далее рекомендуется связать выбранного контрагента с существующим элементом справочника «Контрагенты» в 1С, либо создать нового контрагента.

В первом случае необходимо выбрать контрагента в поле «Контрагент 1С» и нажать кнопку , во втором случае просто нажать кнопку  и дозаполнить необходимые реквизиты в созданном элементе справочника.

Связь аккаунта в ATI.SU и контрагента реализована через дополнительный реквизит контрагента «ATI_DOC_ID» (число):

Для подписанных документов контрагент в первую очередь определяется по сертификату. Если по полям из сертификата не удалось однозначно определить контрагента, то он ищется по дополнительному реквизиту ATI_DOCS_ID. Подробнее о сертификатах и цифровых подписях ниже.

Если выбран тип документа, предполагающий формирование на основе документа 1С, то необходимо выбрать этот документ. При выборе список автоматически фильтруется по организации и контрагенту.

После выбора документа будет автоматически заполнены номер, дата и описание.

Поддерживается отправка файлов объемом не более 10 Мб.

Если отправляемый документ требует согласования со стороны получателя, то выберите пункт «Запросить решение». Тогда у получателя будет возможность утвердить документ или отклонить его (с обоснованием). Обоснование отклонения будет доступно в списке комментариев и выделено красным цветом.

Для сохраненных документов доступна ссылка для скачивания вложенного файла на диск.

Для входящих документов типа 1С («Акт», «Счет») доступна функция формирования печатной формы документа «От имени отправителя».

 

3. Сертификаты и электронные подписи

 

Для обеспечения юридически значимого документооборота необходимо наличие криптографического ПО и сертификата, выданного уполномоченным удостоверяющим центром. В качестве криптографического ПО рекомендуется использовать сертифицированное в соответствии с ФЗ №63, например, КриптоПРО версии не старше 3.6.7777 (https://ru.wikipedia.org/wiki/КриптоПро).

Процедура подписания и отправки документов может быть организована двумя способами: непосредственно в интерфейсе данной обработки или с помощью отдельной утилиты «Агент АТИ-Доки» (далее по тексту - Агент).

Рассмотрим работу без использования Агента. В этом случае сертификаты должны быть установлены локально на компьютере пользователя и указаны в настройках программы и 1С Обработки.
В 1С информация о сертификатах хранится в справочнике «Сертификаты ЭП». Для каждого сертификата, которым предполагается подписывать документы, создается отдельный элемент справочника. При этом сертификат можно импортировать из файла или из хранилища сертификатов. После заполнения всех необходимых реквизитов рекомендуется провести тест сертификата.
Для каждого юридического лица в 1С может использоваться несколько сертификатов, например, с разными периодами действия. В настройках 1С Обработки можно указать для каждой организации свой сертификат.
После заполнения данной таблицы настройки сохраняются автоматически.
Открытый ключ сертификата также передается в составе электронной подписи. По ИНН из свойств сертификата выполняется идентификация юридического лица отправителя.
Если по каким-то причинам установка сертификата на локальные компьютеры невозможна или нежелательна, то можно централизовать процесс подписания и отправки документов с помощью утилиты «Агент АТИ-Доки».

Данная программа устанавливается на компьютер (сервер), куда установлены сертификаты и криптографическое ПО.
Ее можно скачать по адресу (http://d.ati.docs/help/11207599.html).
В результате исполняемый файл будет скопирован в каталог, указанный как рабочий каталог Агента.
В некоторых случаях (например, из-за UAC https://ru.wikipedia.org/wiki/Контроль_учётных_записей_пользователей), программное копирование в выбранных каталог не поддерживается. Тогда следует скопировать файл в папку «Мои документы» и переместить ее в нужный каталог вручную.

После запуска Агента в настройках указывается учетная запись (логин и пароль) и каталог для обмена файлами электронных документов.
Каталог должен быть доступен для чтения и записи под учетными записями пользователей 1С.

 

4. Формирование исходящих документов на основе документов из базы 1С

 

Рассмотрим подробно процедуру формирования электронного документа «Акт» и «Счет».

Для создания электронного документа выполните одно из следующих действий:

  • В меню «Действия» выберите пункт «Отправить новый документ»,
  • Нажмите кнопку «Отправить новый документ», выберите тип документа «Акт» (по умолчанию в формате XML, также поддерживается формат PDF), затем выберите организацию.

Для выбора пользователя «ati.su», которому будет отправляться документ:

  • Введите наименования или число в поле «Получатель (код)».


Если пользователь не найден, происходит переход на сайт «ati.su». Если по выбранному наименованию найдено несколько пользователей, то нужно выбрать необходимого получателя в выпадающем списке.
Если с выбранным пользователем «ati.su» связан элемент справочника «Контрагенты», то он будет автоматически выбран в поле «Контрагент 1С». Если по дополнительному реквизиту «Код_АТИ» контрагент не найден, то поиск ведется по наименованию и ИНН.
Если контрагент в справочнике не найден, создайте новый элемент справочника, либо выберите из существующих.

 

Важно! При создании нового контрагента не происходит проверки на дубликаты среди существующих элементов!


После выбора организации и контрагента выберите сам документ 1С из списка документов «Реализация товаров и услуг». При выборе автоматически устанавливается фильтр по организации и контрагенту. После выбора документа автоматически заполняются поля «Номер», «Дата», «Описание», которые при необходимости можно скорректировать.

Внимание! Идентификатор документа формируется на основании типа документа, внутреннего идентификатора документа (постоянный для каждого документа) и версии данных (меняется автоматически при каждой записи документа). Таким образом, один и тот же документ нельзя отправить 2 раза без перезаписи. Если по какой-то причине документ необходимо отправить повторно, то допускается его перезаписать (это относится и к исправлению ошибок в самом документе 1С) и отправить повторно.

Возможны следующие действия с документом:

  • Отправка контрагенту в виде электронного документа с ЭП непосредственно через «1С Обработку» (нажмите кнопку «Подписать и отправить»). При этом происходит формирование XML-файла на основе документа 1С, сохранение электронного документа на сервер ATI-Доки, подписание документа и отправка его контрагенту.
  • Отправка контрагенту в виде электронного документа без ЭП (нажмите кнопку «Отправить»).
  • Сохранение документа в «Черновики» (нажмите кнопку «Сохранить»). При этом формируется, но не отправляется XML-файл и электронный документ. Далее документ можно открыть, скорректировать и отправить любым из двух вышеприведенных способов.

5. Формирование документов 1С на основе входящих электронных документов

 

1С Обработка позволяет не только формировать электронные документы на основе документов из базы 1С, но и наоборот формировать документы 1С на основе входящих электронных документов. Для формирования входящего документа в базе 1С необходимо одновременное выполнение следующих условий:

  • Тип документа из списка «Акт», «Счет».
  • Формат вложенного файла «XML».
  • Наличие ЭП контрагента-отправителя.
  • Отсутствие в базе документа с соответствующим идентификатором.

Для формирования на основе такого электронного документа записи в базе 1С откройте его форму и нажмите кнопку «Сформировать документ в базе».

Если поля полученного XML-файла заполнены корректно, то будет сформирован непроведенный документ (например: «Платежное поручение», «Поступление товаров и услуг»), который будет доступен в поле «Документ 1С».
